To build an application capable of summarizing long documents in both English and Malay, I would approach the problem as follows:

1. **Preprocessing Steps:**
   - **Text Cleaning:** Remove irrelevant data such as extra spaces, special characters, and unnecessary metadata.
   - **Language Detection:** Automatically identify the language(s) present in each segment of the document.
   - **Sentence Segmentation:** Split the document into sentences, supporting both English and Malay punctuation and grammar.
   - **Tokenization:** Break sentences into tokens (words/phrases), using tools that handle both languages efficiently.
   - **Handling Code-Switching:** Detect and appropriately segment or flag sections where the language switches between English and Malay within the same sentence or paragraph.

2. **Handling Multilingual Support and Code-Switching:**
   - After segmenting the document (e.g., by sentence or paragraph), use a language identification model to tag each segmented item with an identifier specifying its language.
   - If code-switching is present (segments in multiple languages), ensure the summarization model can intelligently process input containing both English and Malay. Approaches include:
     - Utilizing robust multilingual models (such as mBART, mT5, or GPT-based LLMs trained on multilingual corpora).
     - Optionally, group segments by language, summarize each group separately, and then combine the language-specific summaries.
   - Throughout preprocessing and tokenization, ensure that pipelines are language-aware and preserve language-specific features and nuances for each tagged segment.

3. **Approach for Summarization:**
   - Prefer **abstractive summarization** for readability and fluency, but consider **extractive summarization** as a baseline.
   - Start with pre-trained multilingual models supporting both English and Malay ( Exiting LLM models )
   - Fine-tune on parallel or mixed-language summarization datasets, especially if dealing with code-switched texts.
   - If high quality is critical and data is available, consider training/fine-tuning with our personal custom corpus to make sure it specializes & generalizes well with our intended use case

4. **Evaluation of Summary Quality:**
   - Use **automatic metrics** such as ROUGE, BLEU (for both languages), and BERTScore with multilingual embeddings to quantitatively compare summaries to references.
   - Perform **human evaluation** for fluency, coherence, accuracy, and faithfulness in both English and Malay—ideally with bilingual annotators.

By following these steps, I would ensure the summarization system is robust, effective, and provides high-quality results across both English and Malay documents, including those with code-switching. It's important to note that this can be achieved using existing large language models (LLMs), as many current LLMs already support the languages commonly spoken in Malaysia, including both English and Malay.
